Linkedin Audience Network iOS SDK

1st Party Release (Voyager)

  1. Check the last release version to determine the next release version desired.
  2. In terminal go to audience-network-ios and run fastlane ios first_party_release version:*.*.*
  3. Publish Pull Request, Get Reviews, & Merge
  4. Get the new MP Version published from the post-merge job on your pull request
  5. Update Voyager’s Podfile to bump LinkedinAudienceNetworkInternal to the new MP Version

Note: 1st Party Release has two version definitions

  • MP Version: The version of that is automatically managed by the MP and updated when merging a pull request in LinkedinAudienceNetworkInternal - This is what Voyager’s podfile references
  • SDK Version: The version defined in fastlane ios first_party_release version:*.*.* which is used by telemetry to identify clients with a specific version

3rd Party Release

  1. Check the last released version to determine the next release version
  2. Run Workflow with desired release version
